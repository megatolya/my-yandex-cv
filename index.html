1. Инструменты и приёмы для отладки кода

Во всех современных браузерах есть отладочная консоль, а для Firefox есть рекрасный firebug. Я использую только встроенную консоль и всеми любимый alert(), пока мне этого хватало. Но также наслышан о Script Debugger, который позволяет делать точки останова, чтобы более точно отследит момент ошибки, прямо как в Visual Studio.

2. Клик по ссылке

  • После клика по ссылке браузер отправляет HTTP запрос, указывая в строке запроса саму ссылку, метод запроса, версию протокола.
  • Запрос приходит сначала на DNS сервер, который отвечает за то, чтобы выдать реальный адрес сайта, например, Ему пришло http://yandex.com, а он отвечает 127.0.0.255.
  • После DNS сервера с нашим запросом работает уже непосредственно сам сервер, который и отдает нам документ, в зависимости от запроса.
  • Например при разном GET/POST параметре можно получить разное поведение сервера, также в зависимости, какой браузер указан в заголовке, сервер может прислать различные версии сайта. Тут все зависит от фантазии back-end программиста, как воспользоваться информацией клиента.
